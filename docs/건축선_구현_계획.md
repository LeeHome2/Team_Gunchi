```
[선택된 블록 (대지경계선)]
         ↓

  ┌──── 도로 접촉 변 ────┐     ┌──── 비접촉 변 (인접 대지) ────┐
  │ 건축선 후퇴 적용      │     │ 인접 대지경계선 이격거리 적용  │
  │ (도로폭 기반)         │     │ (용도지역 기반)               │
  └──────────────────────┘     └──────────────────────────────┘
         ↓                              ↓
[건축 가능 영역 산출]
         ↓
[CesiumJS 폴리라인으로 건축선 표시 (기존 지적도 표시 방식 동일)]
```

## 4. 작업 목록

### Task 1: 지적도 WFS에서 인접 필지 지목 기반 도로 판별
```
파일: frontend/components/CesiumViewer.tsx (기존 지적도 로직 확장)

- 선택된 필지 주변의 지적도 WFS 데이터에서 지목(jimok) 속성 확인
- 지목이 "도"(도로)인 인접 필지를 식별
- 대지경계선의 각 변(edge)과 도로 필지의 접촉 여부를 판단
  → 기존 WFS 호출 구조 그대로 활용, 추가 API 라우트 불필요
```

```typescript
// 대지경계선의 각 변(edge)과 도로 필지의 접촉 여부 확인
function findRoadAdjacentEdges(cadastralPolygon, nearbyParcels) {
  const edges = getPolygonEdges(cadastralPolygon)
  const roadParcels = nearbyParcels.filter(p => p.properties.jimok === '도')
  const adjacentEdges = []

  for (const edge of edges) {
    for (const road of roadParcels) {
      if (turf.booleanIntersects(edge, road.geometry)) {
        adjacentEdges.push({ edge, roadParcel: road })
        break
      }
    }
  }

  return adjacentEdges  // 건축선 적용 대상
  // 나머지 변 = 인접 대지경계선 이격거리 적용 대상
}
```

### Task 2: Turf.js 설치 및 지오메트리 연산
```bash
cd frontend
npm install @turf/turf
```

```typescript
import * as turf from '@turf/turf'

// 도로 필지 접촉 판단
turf.booleanIntersects(edge, roadParcelPolygon)

// 거리 계산
turf.distance(point1, point2, { units: 'meters' })
```

> ⚠️ **주의**: `turf.buffer(polygon, -distance)`(음수 offset)는 공식 지원되지 않아 불안정할 수 있음.
> 건축선 내부 offset은 백엔드 Shapely의 `polygon.buffer(-distance)`를 사용하는 것을 권장 (이미 의존성에 포함).
> 또는 클라이언트에서 각 edge를 개별적으로 내부로 평행이동하는 방식 사용.

## 6. 데이터 구조

### 건축선 계산 결과
```typescript
interface BuildingLineResult {
  // 원본 대지경계선
  cadastralPolygon: GeoJSON.Polygon

  // 접촉 도로 정보
  adjacentRoads: {
    roadId: string
    roadWidth: number  // 미터
    contactEdge: GeoJSON.LineString
  }[]

  // 용도지역
  zoneType: '일반주거' | '상업' | '공업' | '녹지' | ...

  // 이격거리
  setbacks: {
    fromBuildingLine: number  // 건축선 기준 (m)
    fromAdjacentLot: number   // 인접대지 기준 (m)
  }

  // 계산된 건축선
  buildingLine: GeoJSON.Polygon

  // 건축 가능 영역
  buildableArea: GeoJSON.Polygon
}
```

## 10. 우선순위

1. **[HIGH]** 지적도 WFS 지목 기반 도로 필지 판별 (기존 API 확장)
2. **[HIGH]** Turf.js 설치 및 edge별 도로 접촉 판단
3. **[HIGH]** 용도지역 레이어 연동 (`lt_c_uq111`) — 이격거리 기준 필수
4. **[HIGH]** 이격거리 테이블 구축 (서울시 기준)
5. **[MEDIUM]** 건축선 offset 계산 (백엔드 Shapely buffer)
6. **[MEDIUM]** 건축선 폴리라인 표시 (기존 CesiumJS 렌더링 재활용)
7. **[LOW]** 도로폭 판별 — 4m 미만 도로 처리 (1차는 기본값 4m 이상 가정)
